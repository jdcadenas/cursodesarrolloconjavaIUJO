1. Clases Abstractas: El Plano Maestro

¿Para qué sirven?

Representan un concepto genérico. No puedes comprar un "Animal", compras un "Perro". En Java, la clase abstracta evita que crees objetos incompletos.

  • Garantía: Obliga a los hijos a implementar métodos específicos.
  • Reutilización: El código común se escribe una sola vez.
public abstract class Usuario {
    protected String nombre;

    public Usuario(String n) { this.nombre = n; }

    // Método común con código
    public void saludar() {
        System.out.println("Accediendo como: " + nombre);
    }

    // Método abstracto (OBLIGATORIO)
    public abstract void permisos();
}
Herencia
Implementando la Abstracción
public class Estudiante extends Usuario {
    public Estudiante(String n) { super(n); }

    @Override
    public void permisos() {
        System.out.println("Nivel: Solo Lectura.");
    }
}

El Método Main

public static void main(String[] args) {
    Usuario u = new Estudiante("Carlos");
    u.saludar();
    u.permisos();
}
Accediendo como: Carlos
Nivel: Solo Lectura.
2. Interfaces: Contratos de Habilidades

¿Qué las diferencia?

Una clase solo puede tener un padre (Herencia), pero puede tener muchas habilidades (Interfaces).

Aclaratoria: Las interfaces definen el "QUE" puede hacer un objeto, no el "COMO". 
public interface Autenticable {
    boolean login(String pass);
}

public class Admin implements Autenticable {
    @Override
    public boolean login(String p) {
        return p.equals("admin123");
    }
}
Ejecución de Contratos
public static void main(String[] args) {
    Autenticable auth = new Admin();
    boolean exito = auth.login("admin123");
    System.out.println("¿Acceso concedido? " + exito);
}
$ java PruebaInterfaces
> ¿Acceso concedido? true

Fíjate que usamos el tipo de la interfaz Autenticable para crear la variable. ¡Esto es polimorfismo!

3. Excepciones: El Muro de Seguridad

Manejo de Errores

En lugar de que el programa "explote" (crash), capturamos el error y damos una respuesta amigable.

  • Try: Intenta ejecutar código.
  • Catch: Captura el fallo.
  • Throw: Lanza un error manualmente.
public class Sistema {
    public void validarCupo(int inscritos) throws Exception {
        if (inscritos >= 30) {
            throw new Exception("Sección llena");
        }
        System.out.println("Inscripción exitosa");
    }
}
Atrapando Excepciones en el Main
public static void main(String[] args) {
    Sistema s = new Sistema();
    try {
        s.validarCupo(35); // Esto lanzará el error
    } catch (Exception e) {
        System.err.println("ALERTA: " + e.getMessage());
    }
}
$ java PruebaSistema
> ALERTA: Sección llena
4. Expresiones Regulares (Regex)

Validación Estricta

¿Cómo sabemos si un correo es del IUJO? Regex permite definir patrones de texto exactos.

Patrón: ^[a-z]+@iujo\.edu\.ve$ 
public class ValidadorIUJO {
    public static boolean esEmailValido(String mail) {
        // Estructura: texto + @iujo.edu.ve
        return mail.matches("^[\\w.-]+@iujo\\.edu\\.ve$");
    }
}
Probando el Filtro IUJO
public static void main(String[] args) {
    String m1 = "estudiante@iujo.edu.ve";
    String m2 = "hacker@gmail.com";

    System.out.println(m1 + " ¿Es válido? " + ValidadorIUJO.esEmailValido(m1));
    System.out.println(m2 + " ¿Es válido? " + ValidadorIUJO.esEmailValido(m2));
}
$ java PruebaRegex
> estudiante@iujo.edu.ve ¿Es válido? true
> hacker@gmail.com ¿Es válido? false
5. Genéricos: Moldes Universales

Flexibilidad Tipo-Segura

¿Y si queremos una lista que guarde Estudiantes, pero luego otra que guarde Profesores sin repetir código? Usamos T (Tipo).

import java.util.*;

public class Caja<T> {
    private T contenido;
    
    public void meter(T c) { this.contenido = c; }
    public T sacar() { return contenido; }
}
Ejecución con Diferentes Tipos
public static void main(String[] args) {
    Caja<String> cajaTexto = new Caja<>();
    cajaTexto.meter("Certificado IUJO");
    
    Caja<Integer> cajaNota = new Caja<>();
    cajaNota.meter(20);

    System.out.println("Sacando: " + cajaTexto.sacar());
    System.out.println("Nota final: " + cajaNota.sacar());
}
$ java PruebaGenericos
> Sacando: Certificado IUJO
> Nota final: 20
Reto: Polimorfismo en el Zoológico

Consigna del Reto

Diseña una jerarquía de clases para demostrar el polimorfismo.

  1. Crea una clase abstracta Animal con un método abstracto hacerSonido().
  2. Crea subclases concretas como Perro, Gato y Ave que hereden de Animal.
  3. Implementa hacerSonido() en cada subclase para que devuelva un string único (ej: "Guau", "Miau", "Pío").
  4. En una clase principal, crea una lista de tipo Animal y añade objetos de Perro, Gato y Ave.
  5. Recorre la lista con un bucle y llama al método hacerSonido() de cada objeto.
Solución: Reto del Zoológico
import java.util.ArrayList;
import java.util.List;

abstract class Animal {
    public abstract String hacerSonido();
}

class Perro extends Animal {
    @Override
    public String hacerSonido() { return "Guau"; }
}

class Gato extends Animal {
    @Override
    public String hacerSonido() { return "Miau"; }
}

class Ave extends Animal {
    @Override
    public String hacerSonido() { return "Pío"; }
}

public class Zoologico {
    public static void main(String[] args) {
        List<Animal> animales = new ArrayList<>();
        animales.add(new Perro());
        animales.add(new Gato());
        animales.add(new Ave());

        for (Animal animal : animales) {
            System.out.println("El animal hace: " + animal.hacerSonido());
        }
    }
}
