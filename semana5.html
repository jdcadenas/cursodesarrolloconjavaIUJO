Modularidad: Organización por Paquetes

¿Por qué paquetes?

En proyectos profesionales, no todo va en un solo archivo. Dividimos el código por responsabilidades:

src/
└── com.iujo.gestion/
    ├── main/ (App Principal)
    ├── model/ (Persona, Estudiante)
    ├── service/ (Gestor Genérico)
    └── exception/ (Excepciones Personalizadas)

Control de Visibilidad

  • public: Accesible desde cualquier paquete.
  • protected: Solo heredados y mismo paquete.
  • private: Solo dentro de la misma clase.
Ingeniería: Esta división permite que múltiples programadores trabajen en el mismo proyecto sin interferir entre sí.
Paso 1: Herencia y Encapsulamiento
// Ubicado en com.iujo.gestion.model
public abstract class Persona {
    protected String nombre;
    protected String cedula;

    public Persona(String nombre, String cedula) {
        this.nombre = nombre; this.cedula = cedula;
    }
    public abstract void mostrarPerfil();
    public String getCedula() { return cedula; }
}

public class Estudiante extends Persona {
    private double balanceCuenta;

    public Estudiante(String n, String c) { super(n, c); }

    public void pagarInscripcion(double monto) throws Exception {
        if (monto > balanceCuenta) 
            throw new Exception("Saldo insuficiente ($" + balanceCuenta + ")");
        balanceCuenta -= monto;
    }
    public void recargar(double m) { balanceCuenta += m; }
    public void mostrarPerfil() { 
        System.out.println("[ID: " + cedula + "] " + nombre + " | Saldo: $" + balanceCuenta);
    }
}
Paso 2: Colecciones y Genéricos

Poder de Reutilización

La clase GestorIUJO no sabe qué guarda, solo sabe cómo administrar una lista.

Dato Pro: Usar <T extends Persona> asegura que nuestro gestor solo acepte objetos que sean personas. 
import java.util.ArrayList;

// Ubicado en com.iujo.gestion.service
public class GestorIUJO<T> {
    private ArrayList<T> lista = new ArrayList<>();

    public void agregar(T item) {
        lista.add(item);
    }

    public ArrayList<T> obtenerTodos() {
        return lista;
    }

    public int contar() { return lista.size(); }
}
Paso 3: Menú con Funcionalidades Completas
import java.util.Scanner;

public class AppIUJO {
    private static GestorIUJO<Estudiante> db = new GestorIUJO<>();
    private static Scanner sc = new Scanner(System.in);

    public static void main(String[] args) {
        int opcion = 0;
        do {
            System.out.println("\n--- SISTEMA IUJO (Total: " + db.contar() + ") ---");
            System.out.print("1. Nuevo Estudiante\n2. Cargar Saldo\n3. Pagar Matrícula\n4. Listar\n5. Salir\nOpción: ");
            opcion = sc.nextInt();
            procesar(opcion);
        } while (opcion != 5);
    }

    private static void procesar(int op) {
        try {
            switch(op) {
                case 1: registrar(); break;
                case 2: gestionarSaldo(true); break;
                case 3: gestionarSaldo(false); break;
                case 4: for(Estudiante e : db.obtenerTodos()) e.mostrarPerfil(); break;
            }
        } catch (Exception e) { System.err.println("ERROR: " + e.getMessage()); }
    }
    // ... (métodos registrar() y gestionarSaldo() a continuación)
}
Paso 4: Implementación de Acciones
private static void registrar() {
    System.out.print("Nombre: "); String n = sc.next();
    System.out.print("Cédula: "); String c = sc.next();
    db.agregar(new Estudiante(n, c));
}

private static void gestionarSaldo(boolean esCarga) throws Exception {
    System.out.print("Cédula del estudiante: "); String id = sc.next();
    Estudiante encontrado = null;
    for(Estudiante e : db.obtenerTodos()) {
        if(e.getCedula().equals(id)) encontrado = e;
    }
    
    if(encontrado == null) throw new Exception("Estudiante no registrado");

    System.out.print("Monto $: "); double m = sc.nextDouble();
    if(esCarga) encontrado.recargar(m);
    else encontrado.pagarInscripcion(m);
    System.out.println("¡Operación exitosa!");
}

  • Open/Closed: El sistema está abierto a extenderse (puedo agregar Profesor) pero cerrado a modificar el núcleo.
El Poder de la Clase Object

Sobrescritura de toString()

Por defecto, imprimir un objeto muestra su referencia. Sobrescribir toString nos da una representación legible.

// Dentro de la clase Estudiante
@Override
public String toString() {
    return "Estudiante[" +
           "Cédula=" + cedula +
           ", Nombre=" + nombre + ']';
}

Sobrescritura de equals()

El operador == compara referencias, no contenido. Para comparar objetos por su valor (ej. por cédula), usamos equals.

// Dentro de la clase Estudiante
@Override
public boolean equals(Object obj) {
    if (this == obj) return true;
    if (obj == null || getClass() != obj.getClass()) return false;
    Estudiante other = (Estudiante) obj;
    return cedula.equals(other.cedula);
}
Buenas Prácticas: Control de Versiones

¿Qué es Git?

Es un sistema que guarda un historial de "fotografías" de tu código en el tiempo. Es el estándar de la industria.

  • Permite regresar a versiones anteriores.
  • Facilita el trabajo en equipo.
  • Provee un respaldo de tu trabajo.

Comandos Esenciales

El ciclo básico de trabajo en Git:

# 1. Inicia el repositorio (solo una vez)
$ git init

# 2. Prepara todos tus archivos nuevos/modificados
$ git add .

# 3. Guarda la "fotografía" con un mensaje
$ git commit -m "Mi primer avance del proyecto"
