1. Interactuando: Clase Scanner

Escuchando al Usuario

Para que un programa sea dinámico, debe procesar datos del exterior. Java utiliza la clase Scanner del paquete java.util.

Aclaratoria: System.in representa el flujo de entrada estándar (teclado). Scanner actúa como un intérprete de ese flujo. 
import java.util.Scanner;

public class Entrada {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);
        System.out.print("Nombre: ");
        String n = sc.nextLine();
        System.out.println("Hola, " + n);
    }
}
2. Clase vs. Objeto

El Concepto del Molde

Clase: Es el plano, el concepto abstracto (ej. Vehículo).

Objeto: Es la instancia física, algo que existe en memoria (ej. El carro rojo de Juan).

Ingeniería: Una clase no ocupa espacio en el "Heap" de memoria; el objeto sí. 
public class Vehiculo {
    // Atributos (Estado)
    String marca;
    int velocidad;

    // Métodos (Comportamiento)
    void acelerar() {
        velocidad += 10;
        System.out.println("Acelerando...");
    }
}
Clase y Objeto
Dándole Vida a los Objetos
public class PruebaVehiculo {
    public static void main(String[] args) {
        Vehiculo miCarro = new Vehiculo();
        miCarro.marca = "Toyota";
        miCarro.acelerar();
        
        System.out.println("Marca: " + miCarro.marca);
        System.out.println("Velocidad: " + miCarro.velocidad);
    }
}
Acelerando...
Marca: Toyota
Velocidad: 10
3. El Constructor: Momento del Nacimiento

Reglas de Oro

  • Se llama igual que la clase.
  • No tiene tipo de retorno (ni siquiera void).
  • Se ejecuta automáticamente al usar new.
Aclaratoria: Si no escribes uno, Java crea uno vacío por ti. Pero si escribes uno con parámetros, el vacío desaparece. 
public class Vehiculo {
    String marca;

    // Constructor personalizado
    public Vehiculo(String m) {
        this.marca = m;
        System.out.println("Vehículo creado");
    }
}
4. Sobrecarga: Múltiples Formas

Polimorfismo Estático

Es tener el mismo nombre de método pero con diferentes parámetros. Java decide cuál usar según los datos que envíes.

void frenar() {
    velocidad = 0;
}

void frenar(int intensidad) {
    velocidad -= intensidad;
}
Sobrecarga y Sobreescritura
Probando la Sobrecarga
public static void main(String[] args) {
    Vehiculo v = new Vehiculo("Ford");
    v.velocidad = 100;
    
    v.frenar(30); // Usa la versión con int
    System.out.println("Velocidad tras frenado parcial: " + v.velocidad);
    
    v.frenar(); // Usa la versión sin parámetros
    System.out.println("Velocidad tras frenado total: " + v.velocidad);
}
Vehículo creado
Velocidad tras frenado parcial: 70
Velocidad tras frenado total: 0
5. Encapsulamiento: La Caja Negra

Proteger la Integridad

Ningún objeto debería dejar que toquen sus atributos directamente. Para eso usamos modificadores de acceso.

  • private: Nadie de afuera lo ve.
  • public: Acceso total.
Aclaratoria: Los Getters y Setters son los porteros que validan quién entra y quién sale. 
private int nota;

public void setNota(int n) {
    if(n >= 0 && n <= 20) {
        this.nota = n;
    } else {
        System.out.println("Error: Nota inválida");
    }
}
Encapsulamiento
Estándar JavaBean
public class Alumno {
    private String nombre;

    // Constructor vacío obligatorio
    public Alumno() {}

    // Getter
    public String getNombre() { return nombre; }

    // Setter
    public void setNombre(String n) { this.nombre = n; }
}

¿Por qué este estándar?

Las librerías y frameworks modernos de Java (como Spring o Hibernate) esperan que tus clases sigan esta estructura para poder automatizar procesos.

Probando la Seguridad
public static void main(String[] args) {
    Alumno a = new Alumno();
    a.setNombre("Maria");
    
    // System.out.println(a.nombre); // ERROR: Es privado
    System.out.println("Alumno registrado: " + a.getNombre());
}
$ java PruebaSeguridad
> Alumno registrado: Maria
Reto Final: Clase Empleado

Consigna del Reto

Crear una clase Empleado que encapsule los datos de un trabajador.

  • Atributos privados: nombre (String), salario (double), cargo (String).
  • Constructor: Debe inicializar todos los atributos.
  • Métodos Públicos:
    • Getters y Setters para todos los atributos.
    • aumentarSalario(double porcentaje): Aumenta el salario actual según un porcentaje.
    • mostrarInformacion(): Imprime en consola los detalles del empleado.
¡Aplica todo lo visto en la semana: clases, objetos, constructores y encapsulamiento!
Solución Reto: Empleado
public class Empleado {
    private String nombre;
    private double salario;
    private String cargo;

    public Empleado(String nombre, double salario, String cargo) {
        this.nombre = nombre;
        this.salario = salario;
        this.cargo = cargo;
    }

    public void aumentarSalario(double porcentaje) {
        if (porcentaje > 0) {
            this.salario += this.salario * (porcentaje / 100);
        }
    }

    public void mostrarInformacion() {
        System.out.println("--- Ficha de Empleado ---");
        System.out.println("Nombre: " + this.nombre);
        System.out.println("Cargo: " + this.cargo);
        System.out.println("Salario: $" + String.format("%.2f", this.salario));
    }
    
    // Getters y Setters...
}

public class TestEmpleado {
    public static void main(String[] args) {
        Empleado emp = new Empleado("Ana", 2500.0, "Desarrolladora");
        emp.mostrarInformacion();
        emp.aumentarSalario(10); // Aumento del 10%
        System.out.println("\nSalario actualizado:");
        emp.mostrarInformacion();
    }
}
